# 📝 CHANGELOG - История изменений

## 🎯 Сессия 2026-04-28

### ✅ ЧТО БЫЛО СДЕЛАНО:

#### 1️⃣ **Разделение заказов по типам товаров (мясо/продукты)**
- ✓ Добавлены вкладки в ChefView для переключения между "Продукты" и "Мясо"
- ✓ Шефповара теперь может отправлять списки независимо (только мясо ИЛИ только продукты)
- ✓ Каждый список отправляется отдельно с фильтрацией по категориям
- ✓ Автоматическое определение типа заказа на основе содержимого (функция `_determine_order_type` в CRUD)
- ✓ Установка флагов `sentToMeatSupplier` / `sentToProductSupplier`

#### 2️⃣ **Визуальное отображение типов заказов**
- ✓ Добавлены иконки в интерфейс снабженца:
  - 🥬 Зелёный — только продукты
  - 🥩 Красный — только мясо
  - 🛒 Оранжевый — мясо + продукты одновременно
- ✓ Различие помогает снабженцу и поставщикам быстро определить тип заказа

#### 3️⃣ **Итоговая сумма товаров**
- ✓ Добавлен расчёт и отображение итоговой суммы в режиме обзора
- ✓ Сумма показывается в ChefView и SnabjenecDetailView
- ✓ Формат: млн (для узбекских сумм)

#### 4️⃣ **Поддержка дробных чисел**
- ✓ Работает с точкой (1.5) и запятой (1,5)
- ✓ Система автоматически преобразует запятую в точку
- ✓ Отображение для пользователя — всегда с запятой (локаль)
- ✓ Покрывает все поля ввода количества (ChefView, SnabjenecDetailView)

#### 5️⃣ **Фильтрация добавления товаров**
- ✓ Снабженец может добавить доп. товар ТОЛЬКО из той же категории что и заказ
- ✓ Если заказ только мясо → может добавить только мясо
- ✓ Если заказ только продукты → может добавить только продукты
- ✓ Модальное окно показывает какие категории доступны
- ✓ Попытка добавить товар другой категории → получает ошибку с объяснением

#### 6️⃣ **Блокировка цен после отправки**
- ✓ Цены редактируются ТОЛЬКО в режиме `review_snabjenec` (до отправки)
- ✓ После отправки поставщику (статус `sent_to_supplier`) → цены становятся read-only
- ✓ Визуально: серые, задизейблены
- ✓ Поставщик получает список с финальными ценами

#### 7️⃣ **Синхронизация статусов**
- ✓ Удалены неиспользуемые статусы ('supplier_collecting', 'supplier_delivering', 'completed')
- ✓ Остаются только реальные: 'sent_to_chef', 'review_snabjenec', 'sent_to_supplier', 'waiting_snabjenec_receive', 'sent_to_financier', 'archived'
- ✓ Фронт и бэк теперь синхронизированы

#### 8️⃣ **Маршрут доставки товаров**
- ✓ Полный путь: Шеф → Снабженец → [Мясной поставщик | Продуктовый поставщик] → Снабженец → Финансист
- ✓ Логика различает мясо и продукты на каждом этапе
- ✓ Новые шаблоны готовы к использованию:
  - `SCHOOL Акт приема-передачи Продуктов для Кухни(1).docx`
  - `LAND Акт приема-передачи Продуктов для Кухни(1).docx`
  - `Акт приема-передачи Мяса.docx`
